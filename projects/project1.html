MVCCore_Project: Complete Order Management System

A full-featured web application for managing products, customers, and orders with granular permission-based authorization

February 2026
Solo Project
ASP.NET Core 8 / MVC / EF Core
View Code Live Demo

Overview

A complete business management solution built with ASP.NET Core 8 MVC. The system enables businesses to manage their product inventory, track customer information, and process orders with real-time stock validation. What makes this project unique is its granular permission-based authorization system that allows administrators to control exactly what each user can see and do—down to individual actions like "Create Product" or "Delete Order."

Technologies Used

ASP.NET Core 8 MVC Pattern Entity Framework Core 8 SQL Server ASP.NET Core Identity jQuery / AJAX Bootstrap 5 Newtonsoft.Json Database-First Approach Permission-Based Authorization

The Challenge

The main challenge was implementing a permission system that goes beyond simple role-based authorization. I needed to create a system where administrators could assign specific permissions to individual users—for example, allowing a user to view products but not create them, or create orders but not delete them. This required a custom claims-based permission system that works seamlessly with Identity's built-in authentication.

Solution & Features

I built a comprehensive MVC application with a clean separation of concerns. The solution uses ASP.NET Core Identity for authentication, extended with a custom permission system that stores user permissions as claims. The UI dynamically adapts based on user permissions—menu items and action buttons appear or disappear based on what the logged-in user is allowed to do.

Key Features:

  • Granular Permission System: Custom claims-based authorization where each action (View, Create, Edit, Delete) has its own permission that can be assigned to individual users via checkbox interface
  • Complete Product Management: Full CRUD operations with image upload, category filtering, real-time stock tracking, and active/inactive status toggling
  • Intelligent Order Processing: Create and edit orders with dynamic item addition/removal, automatic customer creation/updating, and real-time stock validation before saving
  • Database-First with Migrations: Successfully integrated with existing database schema while adding Identity tables and managing schema changes without data loss
  • Responsive UI: Clean, modern interface built with Bootstrap that works seamlessly on desktop and mobile devices
  • Transaction Management: All order operations wrapped in database transactions to ensure data integrity during stock updates

Code Highlight: Custom Permission System

One of the most challenging and rewarding parts of this project was implementing the granular permission system. Here's how I created a custom authorization attribute that checks for specific permissions: 

// Custom permission attribute
public class PermissionAuthorizeAttribute : AuthorizeAttribute
{
    public PermissionAuthorizeAttribute(string permission)
    {
        Policy = permission; // Policy name is the permission itself
    }
}

// Usage in controller
[PermissionAuthorize(Permissions.ProductCreate)]
public async Task<IActionResult> Create()
{
    // Only users with explicit "Product.Create" permission can access
    return View();
}

// Dynamic policy registration in Program.cs
builder.Services.AddAuthorization(options =>
{
    foreach (var permission in Permissions.GetAllPermissions())
    {
        options.AddPolicy(permission, policy =>
            policy.Requirements.Add(new PermissionRequirement(permission)));
    }
});

This system allows administrators to assign permissions via a checkbox interface, and the UI automatically adapts—buttons and menu items only appear for users with the corresponding permissions.

What I Learned

This project taught me invaluable lessons about building production-ready web applications:

  • EF Core Migrations: Deep understanding of managing database schema changes while preserving existing data, including handling foreign key constraints and proper drop ordering
  • Custom Authorization Systems: How to extend ASP.NET Core Identity beyond simple roles to create granular, permission-based access control
  • Transaction Management: Ensuring data integrity with database transactions, especially critical for order processing that involves multiple related tables
  • Client-Side State Management: Maintaining complex UI state in JavaScript for dynamic order item management before server submission
  • Problem-Solving: Debugging migration conflicts and learning to manually manage the migration history table when needed

I'm particularly proud of the permission system—it transforms a standard CRUD application into a flexible business tool where administrators have complete control over user access without any code changes.
